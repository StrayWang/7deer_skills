# gefei-seo-monetization — 哥飞流量变现体系

## 核心理念

哥飞方法论的核心是「流量不要钱」——通过 SEO 免费流量 + Adsense 广告变现，构建低成本、可持续的出海盈利模型。

**流量四板斧**（按优先级排序）：
1. **SEO 免费流量** — 长期稳定，成本趋近于零，优先级最高
2. **各大平台广告投流** — 短期快速获客，ROI 需严格把控
3. **红人传播** — 社交裂变，适合有一定产品力的阶段
4. **发帖灌水** — Reddit / Quora / 论坛，低成本长尾流量补充

## 第一阶段：关键词选品

**哥飞核心原则：一个关键词一个页面**

```
输入：你的网站方向 / 目标用户群体
输出：可执行的关键词清单（含竞争度评分）
```

### 步骤 1：用免费工具组合挖词

```python
# 工具组合（全部免费版可用）
tools = {
    "Google Trends": "发现上升趋势词，捕捉时间窗口",
    "Google Search Console": "挖掘已有曝光但排名靠后的词（哑炮词抢救）",
    "AnswerThePublic": "挖掘用户真实提问词",
    "Similarweb（免费版）": "分析竞品流量来源词",
    "Ahrefs（免费版）": "检查词的 KD（关键词难度）",
}

# 哥飞筛词标准
criteria = {
    "搜索量": "月搜索量 100~10000（避开头部红海）",
    "竞争度 KD": "< 30（新站优先选 KD < 15）",
    "商业意图": "Informational / Transactional 均可，Adsense 站偏 Informational",
    "长尾程度": "3~5 词的长尾词优先，竞争小、意图明确",
}
```

### 步骤 2：哑炮词抢救（哥飞独门技法）

```
在 Google Search Console → 性能报告 中：
- 筛选：展示次数 > 100，点击次数 < 10
- 这些词 = 谷歌认为你相关，但标题/描述不够吸引人
- 操作：用 AI 重写标题，加入更强的钩子词
  例：「减肥食谱」→「7天瘦5斤的懒人食谱（不用运动）」
```

### 步骤 3：关键词可行性判断（哥飞五维评估）

```
评估维度：
1. 搜索量是否足够（工具：GSC / Ahrefs Free）
2. 竞争对手实力（查 SERP 前10，是否有大站霸榜）
3. 内容可生产性（你能写出比现有结果更好的内容吗）
4. 变现路径（Adsense CPC 高不高？有没有联盟产品可推）
5. 时间窗口（Google Trends 是上升还是下降趋势）

评分：5 维都满足 → 立刻做；3~4 维满足 → 可做；< 3 维 → 跳过
```

## 第三阶段：Adsense 变现公式

```
月收入 = 日均 PV × Adsense RPM × 30

其中：
- RPM（千次展示收益）= 广告 CPC × CTR × 1000
- 不同类目 RPM 参考：
  金融/保险：$15~50
  AI 工具/科技：$5~15
  游戏/娱乐：$1~5
  中文内容：$0.5~2（出海做英文站的核心原因）

优化 RPM 的方法：
1. 广告位置：首屏内容中插 > 侧边栏 > 底部
2. 广告密度：内容每 300~500 字插一个，不超过内容的 30%
3. 类目选择：优先选高 CPC 类目（金融、法律、健康、AI）
4. 页面速度：Core Web Vitals 全绿，RPM 可提升 20~40%
```

## 第四阶段：付费转化升级（Adsense → 订阅）

```
哥飞路径：Adsense 养站 → 积累用户 → 推出付费功能

转化漏斗：
免费流量 → 免费工具使用 → 邮件订阅 → 付费升级

关键指标：
- 免费→付费转化率目标：2~5%
- 邮件打开率目标：> 25%
- 付费留存率（月）目标：> 70%

工具推荐：
- 邮件收集：Resend / ConvertKit
- 支付：Stripe（国际）/ Lemon Squeezy（含税务处理）
- 分析：PostHog（免费开源）
```

```python
# 哑炮词标题优化脚本
# 文件路径：site-keyword-research/scripts/rescue_dead_keywords.py

import anthropic
import csv
import json

client = anthropic.Anthropic()

TITLE_REWRITE_PROMPT = """
你是一位 SEO 专家，擅长撰写高点击率的页面标题。

当前页面信息：
- 原标题：{original_title}
- 目标关键词：{keyword}
- 当前展示次数：{impressions}
- 当前点击率：{ctr}%
- 竞争对手标题（SERP Top 3）：{competitor_titles}

请生成 3 个优化版标题，要求：
1. 关键词放在标题前半段
2. 包含数字或具体利益点
3. 长度 50~60 字符（英文）
4. 避免标题党，保持真实性

输出 JSON 格式：
{{"titles": ["标题1", "标题2", "标题3"], "reasoning": "优化逻辑说明"}}
"""

def rescue_dead_keywords(input_csv: str, output_json: str):
    """
    输入：从 GSC 导出的哑炮词 CSV
    输出：优化后的标题建议 JSON
    """
    results = []
    
    with open(input_csv, 'r', encoding='utf-8') as f:
        reader = csv.DictReader(f)
        for row in reader:
            if float(row.get('Impressions', 0)) > 100 and float(row.get('CTR', '0%').replace('%', '')) < 5:
                prompt = TITLE_REWRITE_PROMPT.format(
                    original_title=row.get('Page Title', ''),
                    keyword=row.get('Query', ''),
                    impressions=row.get('Impressions', ''),
                    ctr=row.get('CTR', '').replace('%', ''),
                    competitor_titles="（请手动填入 SERP 前3标题）"
                )
                
                message = client.messages.create(
                    model="claude-opus-4-5",
                    max_tokens=500,
                    messages=[{"role": "user", "content": prompt}]
                )
                
                response_text = message.content[0].text
                try:
                    suggestion = json.loads(response_text)
                except json.JSONDecodeError:
                    suggestion = {"titles": [response_text], "reasoning": "解析失败，请手动处理"}
                
                results.append({
                    "keyword": row.get('Query', ''),
                    "original_title": row.get('Page Title', ''),
                    "impressions": row.get('Impressions', ''),
                    "current_ctr": row.get('CTR', ''),
                    "suggested_titles": suggestion.get('titles', []),
                    "reasoning": suggestion.get('reasoning', '')
                })
    
    with open(output_json, 'w', encoding='utf-8') as f:
        json.dump(results, f, ensure_ascii=False, indent=2)
    
    print(f"✅ 处理完成，共 {len(results)} 个哑炮词，结果已保存到 {output_json}")
    return results


if __name__ == "__main__":
    import sys
    if len(sys.argv) < 3:
        print("用法：python rescue_dead_keywords.py input.csv output.json")
    else:
        rescue_dead_keywords(sys.argv[1], sys.argv[2])

import json
from dataclasses import dataclass
from typing import List, Optional

@dataclass
class RequirementsDoc:
    """刘小排需求文档标准格式"""

# 上下文管理器：防止 Agent 在长对话中「失忆」
# 文件路径：python-agent-engine/scripts/context_manager.py

import json
from pathlib import Path
from datetime import datetime

class AgentContextManager:
    """
    刘小排「代码屎山预防」上下文管理器
    
    核心思路：
    - 每个功能模块独立的上下文文件
    - 定期生成「当前状态摘要」供 Agent 快速恢复上下文
    - 关键决策记录存档，避免 Agent 反复询问同一问题
    """
    
    def __init__(self, project_name: str, context_dir: str = ".agent/context"):
        self.project_name = project_name
        self.context_dir = Path(context_dir)
        self.context_dir.mkdir(parents=True, exist_ok=True)
        
    def save_decision(self, decision_type: str, content: dict):
        """记录关键决策（技术选型、架构决定等）"""
        decisions_file = self.context_dir / "decisions.json"
        
        decisions = []
        if decisions_file.exists():
            with open(decisions_file) as f:
                decisions = json.load(f)
        
        decisions.append({
            "timestamp": datetime.now().isoformat(),
            "type": decision_type,
            "content": content
        })
        
        with open(decisions_file, 'w') as f:
            json.dump(decisions, f, ensure_ascii=False, indent=2)
        
        print(f"✅ 决策已记录：{decision_type}")
    
    def generate_status_summary(self, completed_tasks: list, current_task: str, blockers: list = None) -> str:
        """
        生成当前项目状态摘要
        在每次新开对话时，将此摘要粘贴给 Agent，快速恢复上下文
        """
        summary = f"""
# {self.project_name} — 当前状态摘要
生成时间：{datetime.now().strftime('%Y-%m-%d %H:%M')}

## 已完成
{chr(10).join(f'- ✅ {task}' for task in completed_tasks)}

## 当前任务
- 🔄 {current_task}

## 阻塞项
{chr(10).join(f'- ⚠️  {b}' for b in (blockers or [])) or '- 无'}

## 关键决策记录
"""
        decisions_file = self.context_dir / "decisions.json"
        if decisions_file.exists():
            with open(decisions_file) as f:
                decisions = json.load(f)
            for d in decisions[-5:]:  # 只显示最近5条
                summary += f"- [{d['type']}] {json.dumps(d['content'], ensure_ascii=False)}\n"
        
        # 保存摘要文件
        summary_file = self.context_dir / "latest_summary.md"
        with open(summary_file, 'w', encoding='utf-8') as f:
            f.write(summary)
        
        return summary


if __name__ == "__main__":
    # 使用示例
    ctx = AgentContextManager("AI Logo Generator")
    
    # 记录技术选型决策
    ctx.save_decision("tech_stack", {

    )
    
    print("📋 状态摘要已生成，请在新对话开始时粘贴给 Agent：")
    print(summary)

# 多模型协作路由器（参考刘小排「三模型协作」工作流）

from enum import Enum

class TaskType(Enum):
    LOGIC_HEAVY = "logic"        # 复杂逻辑、长上下文 → Claude
    CODE_REVIEW = "review"       # 代码审查、单元测试 → GPT Codex  
    UI_DESIGN = "ui"             # 界面设计、前端代码 → Gemini

def route_task(task_description: str) -> tuple[TaskType, str]:
    """
    根据任务描述自动路由到最合适的模型
    返回：(任务类型, 推荐模型)
    """
    task_lower = task_description.lower()
    
    ui_keywords = ["界面", "ui", "前端", "样式", "css", "tailwind", "组件", "布局", "设计"]
    review_keywords = ["审查", "review", "测试", "test", "bug", "修复", "检查代码"]
    
    if any(kw in task_lower for kw in ui_keywords):
        return TaskType.UI_DESIGN, "Gemini CLI (gemini-2.5-pro) — 多模态 UI 设计能力强"
    elif any(kw in task_lower for kw in review_keywords):
        return TaskType.CODE_REVIEW, "Codex CLI (gpt-5.3-codex) — 代码审查速度快"
    else:
        return TaskType.LOGIC_HEAVY, "Claude Code (claude-opus-4-5) — 长上下文逻辑能力强"
